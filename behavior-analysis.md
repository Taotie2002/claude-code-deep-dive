# Claude Code 系统行为分析

> 本章分析 Claude Code 运行时各子系统的行为触发机制

---

## 1. Agent Spawn 行为

Agent Spawn 是 Claude Code 实现多代理协作的核心机制。

### 1.1 何时 Spawn 子 Agent

```mermaid
flowchart TD
    A[任务接收] --> B{任务复杂度判断}
    B -->|单任务简单| C[主Agent直接执行]
    B -->|多任务并行| D[并行Spawn]
    B -->|任务嵌套| E[层级Spawn]
    B -->|资源受限| F[单Agent顺序执行]
    
    D --> G[分配独立Session]
    E --> H[建立父子关系链]
    
    G --> I[独立模型响应]
    H --> J[共享Context片段]
```

**触发条件：**

| 条件类型 | 触发标准 | 行为 |
|---------|---------|------|
| 显式调用 | 用户/系统明确请求 | 立即Spawn |
| 隐式推断 | 任务含多独立子目标 | 按需Spawn |
| 溢出检测 | 单一Context超阈值 | Fork新Agent |
| 负载均衡 | 检测到长任务 | Spawn分担 |

### 1.2 Fork 模式

Fork 模式创建独立运行的 Agent 副本：

```mermaid
flowchart LR
    A[父Agent] -->|Fork| B[子Agent-1]
    A -->|Fork| C[子Agent-2]
    A -->|Fork| D[子Agent-N]
    
    B --> E[独立Session]
    C --> F[独立Session]
    D --> G[独立Session]
    
    E --> H[独立模型实例]
    F --> I[独立模型实例]
    G --> J[独立模型实例]
```

**Fork 触发场景：**
- 多文件并行处理（每个文件一个Agent）
- A/B 测试同一任务的两种方案
- 独立工具链验证

### 1.3 进程内模式

进程内模式在同一进程内共享状态：

```mermaid
flowchart TD
    A[主Agent] --> B[子任务队列]
    B --> C{进程模式选择}
    C -->|进程内| D[共享内存空间]
    C -->|独立进程| E[IPC通信]
    
    D --> F[低延迟通信]
    E --> G[进程隔离]
    
    F --> H[状态直接访问]
    G --> H2[序列化传输]
```

**进程内优势：** 零序列化开销，适合高频交互任务

## 2. Context 压缩行为

### 2.1 何时压缩上下文

```mermaid
flowchart TD
    A[每轮交互结束] --> B{Context使用率检测}
    B -->|< 70%| C[无需压缩]
    B -->|70-85%| D[轻度压缩警告]
    B -->|> 85%| E[触发压缩]
    
    E --> F[选择压缩策略]
    F --> G[执行压缩]
    G --> H[验证压缩质量]
    H -->|通过| I[更新Context]
    H -->|失败| J[回滚+重试]
```

### 2.2 阈值触发机制

| 层级 | 阈值 | 压缩强度 | 影响 |
|------|------|---------|------|
| 安全区 | < 70% | 无 | 正常响应 |
| 警戒区 | 70-85% | 选择性 | 优化非关键消息 |
| 危险区 | 85-95% | 强制性 | 删除低价值内容 |
| 紧急区 | > 95% | 激进 | 保留核心指令 |

### 2.3 压缩算法

```mermaid
flowchart LR
    A[原始Context] --> B[内容分析]
    B --> C{内容分类}
    
    C -->|系统指令| D[保留: 100%]
    C -->|用户输入| E[保留: 90%]
    C -->|Tool结果| F[保留: 70%]
    C -->|历史对话| G[保留: 50%]
    C -->|旁路信息| H[保留: 20%]
    
    D --> I[合并相似片段]
    E --> I
    F --> I
    G --> I
    H --> I
    
    I --> J[生成摘要]
    J --> K[压缩后Context]
```

**压缩策略优先级：**
1. 保留系统级关键指令（AGENTS.md、SOUL.md）
2. 保留用户最近 3-5 轮核心输入
3. 保留 Tool 调用结果（压缩非关键参数）
4. 合并相似主题的历史对话
5. 删除已被替代的中间结果

### 2.4 恢复机制

```mermaid
flowchart TD
    A[压缩执行] --> B[生成压缩快照]
    B --> C[存储在Memory层]
    
    D[后续请求] --> E{需要历史内容?}
    E -->|是| F[从快照恢复]
    E -->|否| G[直接使用压缩Context]
    
    F --> H[按需解压]
    H --> I[匹配关键片段]
    I --> J[补全Context]
```

**恢复触发：**
- 用户明确请求历史内容
- 任务依赖之前中间结果
- 调试/复盘场景

## 3. Tool 调用行为

### 3.1 何时调用 Tool

```mermaid
flowchart TD
    A[LLM输出解析] --> B{Tool调用检测}
    B -->|无Tool调用| C[直接返回文本]
    B -->|检测到Tool| D[进入Tool链]
    
    D --> E[权限检查]
    E -->|通过| F[参数验证]
    E -->|拒绝| G[返回权限错误]
    
    F -->|通过| H[执行Tool]
    F -->|失败| I[返回参数错误]
    
    H --> J[结果格式化]
    J --> K[注入Context]
    K --> L[继续LLM处理]
```

### 3.2 权限检查流程

```mermaid
flowchart TD
    A[Tool调用请求] --> B[提取Tool名称]
    B --> C[查询Tool清单]
    C --> D{白名单检查}
    
    D -->|白名单内| E[权限Level检查]
    D -->|不在白名单| F[默认拒绝]
    
    E --> G{权限Level足够?}
    G -->|是| H[安全验证链]
    G -->|否| I[降级处理/拒绝]
    
    H --> J[最终放行]
    F --> K[日志记录]
    I --> K
    J --> L[进入执行阶段]
    
    K --> M[安全审计日志]
```

**权限Level定义：**

| Level | 类型 | 说明 |
|-------|------|------|
| 0 | 禁用 | 完全不可用 |
| 1 | 只读 | 仅查询类Tool |
| 2 | 读写 | 基础文件/网络操作 |
| 3 | 敏感 | 系统配置修改 |
| 4 | 危险 | 破坏性操作 |

### 3.3 安全验证链

```mermaid
flowchart LR
    A[Tool请求] --> B[身份验证]
    B --> C[意图分析]
    C --> D[风险评估]
    D --> E{风险等级}
    
    E -->|低风险| F[快速放行]
    E -->|中风险| G[二次确认]
    E -->|高风险| H[人工审批]
    
    F --> I[执行]
    G --> J{用户确认?}
    J -->|确认| I
    J -->|拒绝| K[终止]
    H --> L{人工批准?}
    L -->|批准| I
    L -->|拒绝| K
    
    I --> M[记录审计日志]
    K --> M
```

## 4. Memory 管理行为

### 4.1 何时读写记忆

```mermaid
flowchart TD
    A[Session启动] --> B[加载持久化Memory]
    B --> C[合并到工作Context]
    
    D[交互进行中] --> E{关键信息检测}
    E -->|是| F[写入临时记忆]
    E -->|否| G[继续交互]
    
    F --> H{信息重要性判断}
    H -->|高| I[标记持久化候选]
    H -->|低| J[仅会话级保留]
    
    K[Session结束] --> L{持久化检查}
    L -->|候选内容| M[写入持久化存储]
    L -->|临时内容| N[清理释放]
    
    M --> O[更新MEMORY.md]
    N --> P[清理临时缓存]
```

### 4.2 会话级 vs 持久化

| 存储类型 | 时机 | 内容 | 生命周期 |
|---------|------|------|---------|
| **会话级** | 实时写入 | 当前任务中间结果 | Session结束 |
| **持久化** | Session结束 | 决策、教训、重要上下文 | 长期保留 |

**会话级记忆触发写入：**
- Tool 调用结果（特别是网络/文件类）
- 任务进度节点完成
- 用户提供的临时偏好

**持久化记忆触发写入：**
- 明确决策点（"用户选择方案A"）
- 错误教训（"上次这样做的坑"）
- 跨任务模式（"用户通常在周末工作"）

### 4.3 自动提取时机

```mermaid
flowchart TD
    A[Session运行中] --> B{自动提取检测}
    
    B --> C{提取触发条件}
    C -->|新技能习得| D[gm_record: SKILL]
    C -->|问题解决| E[gm_record: TASK]
    C -->|重要事件| F[gm_record: EVENT]
    C -->|无触发| G[跳过]
    
    D --> H[知识图谱写入]
    E --> H
    F --> H
    
    H --> I[PageRank更新]
    I --> J[社区检测刷新]
    J --> K[图谱维护完成]
    
    G --> L[继续正常流程]
    K --> L
```

**自动提取规则：**

```
触发条件：
├── Tool调用成功 → 记录为已掌握技能
├── 错误恢复成功 → 记录为解决方案
├── 用户纠正 → 记录为踩坑教训
└── 任务完成 → 记录为成功模式
```

## 5. 行为联动关系

### 5.1 核心行为交互

```mermaid
flowchart TD
    A[用户请求] --> B[Context评估]
    B --> C{需要压缩?}
    C -->|是| D[Context压缩]
    C -->|否| E[继续]
    
    D --> F[检查是否Spawn]
    E --> F
    
    F --> G{需要子Agent?}
    G -->|是| H[Agent Spawn]
    G -->|否| I[主Agent处理]
    
    H --> J[Tool调用]
    I --> J
    
    J --> K{涉及外部操作?}
    K -->|是| L[权限+安全验证]
    K -->|否| M[直接执行]
    
    L --> N[Memory写入]
    M --> N
    
    N --> O[Session结束?]
    O -->|否| B
    O -->|是| P[持久化Memory]
```

### 5.2 行为优先级

```
1. [最高] 安全验证 → 任何操作前必须通过
2. [高] Context管理 → 保证响应能力
3. [中] Tool调用 → 任务执行核心
4. [低] Memory持久化 → 可延迟处理
5. [最低] Agent Spawn → 按需触发
```

## 6. 小结

| 行为系统 | 核心触发 | 关键机制 |
|---------|---------|---------|
| Agent Spawn | 任务并行/复杂度 | Fork/进程内双模式 |
| Context压缩 | 使用率阈值 | 分层压缩+快照恢复 |
| Tool调用 | LLM输出解析 | 多级权限+安全链 |
| Memory管理 | 关键信息检测 | 会话级临时+持久化双层 |

> 行为级分析揭示了 Claude Code 各子系统如何协同工作。理解这些触发机制和交互关系，是进行系统调优和异常排查的基础。
